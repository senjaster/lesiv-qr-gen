# Руководство пользователя - Генератор QR-кодов

## Описание программы

Программа предназначена для автоматической генерации PDF-файлов с уникальными QR-кодами. Каждый QR-код содержит уникальный идентификатор и размещается на странице согласно заданному шаблону.

## Интерфейс программы

После запуска вы увидите окно с несколькими полями для заполнения:

### 1. **Шаблон** (PDF-файл)
- **Что это:** PDF-документ, на который будут наложены QR-коды
- **Как выбрать:** Нажмите кнопку **"Обзор"** справа от поля
- **Что выбирать:** Ваш готовый PDF-шаблон (например, "Лицо Pro90 без кода.pdf")
- **Важно:** Файл должен быть в формате PDF

### 2. **Положение QR-кодов** (CSV-файл)
- **Что это:** Файл который определяет, где именно на странице будут размещены QR-коды
- **Как выбрать:** Нажмите кнопку **"Обзор"** справа от поля
- **Что выбирать:** Файл positions.csv
- **Важно:** Этот файл можно редактировать в Блокноте для изменения расположения QR-кодов

### 3. **Готовые файлы** (папка для результатов)
- **Что это:** Папка, куда программа сохранит готовые PDF-файлы
- **Как выбрать:** Нажмите кнопку **"Обзор"** справа от поля
- **Что происходит:** 
  - Программа автоматически создаст внутри выбранной папки подпапку с текущей датой (например, "2026-03-04")
  - Все готовые файлы будут сохранены в эту подпапку по дате

### 4. **Количество страниц**
- **Что это:** Сколько страниц (файлов) нужно создать
- **Как указать:** Введите число в поле (например, 10, 20, 50)
- **Важно:** 
  - Каждая страница будет сохранена в отдельный PDF-файл
  - Можно генерировать дополнительно: сначала 20 страниц, потом еще 10 и т.д.
  - Каждый файл будет иметь уникальные QR-коды

### 5. **Базовый URL QR-кода**
- **Что это:** Начало ссылки, которая будет закодирована в QR-код
- **Пример:** https://qr.thermoelectrica.com?id=
- **Как это работает:** К этому адресу добавляется уникальный номер, например https://qr.thermoelectrica.com?id=1772572199
- **Важно:** Это поле уже заполнено правильно, менять его не нужно!

### 6. Окно "Сообщения"

В нижней части программы находится окно с сообщениями, где отображается:
- Выбранные файлы
- Процесс генерации
- Информация о каждом созданном файле
- Сообщения об ошибках (если они возникли)



## Дополнительная генерация

Вы можете генерировать файлы несколько раз:

**Пример:**
1. Первый раз: указали 20 страниц → получили файлы с 001 до 020
2. Второй раз: указали 10 страниц → получите файлы с 021 до 030
3. И так далее

**Важно:** 
- Программа запоминает, где остановилась
- Каждый новый файл будет иметь уникальные QR-коды
- Файлы за один день сохраняются в одну папку по дате



## Редактирование расположения QR-кодов

Файл positions.csv можно открыть Блокноте и изменить.

### Структура файла:
```
x_mm,y_mm,size_mm,rotation_deg
143.09,44.79,14.67,0
143.09,116.79,14.67,0
168.66,47.04,14.67,270
```

### Параметры:
- **x_mm** — расстояние от левого края страницы до центра QR-кода (в миллиметрах)
- **y_mm** — расстояние от верхнего края страницы до центра QR-кода (в миллиметрах)
- **size_mm** — размер QR-кода (в миллиметрах)
- **rotation_deg** — поворот QR-кода (0, 90, 180 или 270 градусов)

**Как редактировать:**
1. Откройте positions.csv в Блокноте
2. Измените нужные значения координат
3. Сохраните файл
4. Запустите генерацию заново



## Сохранение настроек

Программа автоматически запоминает:
- Последний выбранный шаблон
- Последний выбранный CSV-файл
- Последнюю папку для результатов
- Количество страниц
- Базовый URL

**При следующем запуске** все поля будут уже заполнены последними значениями.
